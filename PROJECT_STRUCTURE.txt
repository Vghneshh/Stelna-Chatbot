VALIDATION MATRIX (HYBRID - Signal + LLM Fallback)
================================================================================

File: ai/intentAnalyzer.js

CURRENT APPROACH (March 4, 2026):
  Hybrid 2-layer validation: Fast signals + Intelligent LLM fallback

  LAYER 1: Signal Extraction (Fast Path - No API Calls)
    Parse user answer for domain-specific signals
    Q1: Looks for product|problem|user|market signals
    Q2: Looks for timeline|timeframe|milestone|stage signals
    Q3: Looks for design|toolsUsed|appearance|functional signals
    Q4: Looks for functionality|sensors|outputs|control signals
    Q5: Looks for material|manufacturing|sourcing signals
    Q6: Looks for standards|safety|domain signals
    Q7: Looks for testing|userTest|durability signals
    Q8: Looks for safetyAware|mitigation signals
    Q9: Looks for electronics|processor|communication signals
    Q10: Looks for components|documented signals
    
    If ANY relevant signal found → ✅ ACCEPTED (instant, no further processing)

  LAYER 2: LLM Validation (Intelligent Fallback - Only if uncertain)
    If no signals found, ask LLM: "Does this answer reasonably address [question]?"
    LLM understands semantic intent and context
    Results in confident VALID or INVALID decisions
    
    If LLM confirms valid → ✅ ACCEPTED
    If LLM rejects → ❌ INCOMPLETE (asks follow-up)

VALIDATION FLOW:
    User Answer
         ↓
    Extract Signals (fast, no API)
         ├→ Found signals? → ✅ ACCEPT (done)
         └→ No signals? → Continue to LLM
         ↓
    Query LLM (only when needed)
         ├→ Valid? → ✅ ACCEPT
         └→ Invalid? → ❌ REJECT

BENEFITS OF HYBRID APPROACH:
  ✓ SPEED: 80-90% of answers accepted instantly via signals (no API latency)
  ✓ FLEXIBILITY: Intelligent LLM accepts paraphrased/complex answers
  ✓ COST EFFICIENT: API calls only for ambiguous cases (minimal cost)
  ✓ RELIABILITY: Safe fallback if LLM unavailable (signals alone still work)
  ✓ PROGRESSIVENESS: Asks targeted follow-ups for missing information
  
EXAMPLES:

  FAST PATH (signals only):
    Q2: "We plan to launch next summer"
         → Signals: {timeline, timeframe, milestone}
         → ✅ ACCEPTED instantly
         → Cost: $0.00 (no API)
    
    Q4: "It measures water intake and sends alerts"
         → Signals: {sensors, outputs}
         → ✅ ACCEPTED instantly
         → Cost: $0.00 (no API)

  LLM PATH (fallback for edge cases):
    Q8: "Non toxic"
         → Signals: {} (no safety keywords detected)
         → Query LLM: "Does 'non toxic' address safety factors?"
         → LLM: "VALID - implies non-toxic materials are safe"
         → ✅ ACCEPTED
         → Cost: $0.01 (minimal)
    
    Q2: "Smart device"
         → Signals: {} (no timeline indicators)
         → Query LLM: "Does 'smart device' address development timeline?"
         → LLM: "INVALID - no timeline information"
         → ❌ REJECTED, asks for timeline details
         → Cost: $0.01 (minimal)
